API Geotécnica TMO

Screening geotécnico remoto via SoilGrids 250m + SGB/CPRM

Versão 2.0 · Com correção geológica · 03/2026
1. Visão Geral e Propósito

A API Geotécnica TMO é um serviço de triagem geotécnica remota que deriva propriedades de engenharia civil a partir de dados satelitais de solo (SoilGrids 250m) e contexto geológico (SGB/CPRM).

O que a API faz
  • 14 propriedades brutas do solo (argila, areia, silte, pH, CEC, carbono, densidade, água retida, etc.)
  • Contexto geológico da formação rochosa sob aquele ponto (nome, litologia, era, mineralogia inferida)
  • 10 derivações geotécnicas calculadas por funções de pedotransferência (Ksat, erodibilidade, Atterberg, USCS, shrink-swell, compactação, CBR, fundação, comparação de camadas, risco)
  • Correções por mineralogia que ajustam as derivações com base no mineral de argila dominante
  • Score de risco (0–16) com fator de prêmio, flags e checklist de inspeção
  • Nível de confiança (high / medium / low) e disclaimer regulatório (NBR 6122/8036)
O que a API NÃO faz
  • Não substitui investigação geotécnica de campo (SPT, CPT, laboratório)
  • Não fornece dados de resistência à penetração, nível d'água, colapsividade ou agressividade química
  • Não cobre profundidades maiores que 2 metros (SoilGrids vai até 200cm; SPT vai a 18m+)
  • Não fornece valores absolutos para dimensionamento de fundações
Papel no processo TMO

A API opera como primeiro passe de triagem no fluxo de underwriting do seguro quinquenal:

  1. Pré-classificar o terreno antes de enviar equipe de campo
  2. Direcionar a inspeção (o inspetor sabe o que esperar e o que testar)
  3. Precificar o prêmio inicial (ajustado posteriormente pelo laudo de campo)
  4. Documentar a base técnica da decisão de underwriting

Fonte 1: SoilGrids 250m ISRIC via GEE
  • Mapa global de propriedades do solo em resolução de 250m por pixel, produzido pelo ISRIC (Wageningen, Holanda)
  • Modelos de machine learning treinados com ~240.000 amostras de campo globais, usando 400+ covariáveis ambientais
  • 6 camadas padrão (0–5cm até 100–200cm) + compostas por média ponderada
  • Acurácia reportada: R² de 0.3 a 0.7 — melhor em zonas temperadas; pior em tropicais
  • Limitação chave: Não diferencia mineralogia da argila
Fonte 2: SGB/CPRM Litoestratigrafia
  • Mapa geológico do Brasil em escala 1:1.000.000, produzido pelo Serviço Geológico do Brasil
  • Retorna formação geológica, litologia dominante, era geológica
  • A formação geológica determina o mineral de argila dominante → correção
  • Serviço público WFS, sem autenticação. Sujeito a instabilidade (timeout 5s, fallback gracioso)

Resolução e Método
  • Resolução espacial: 250m × 250m por pixel
  • Método: Buffer circular de 2000m ao redor da coordenada, média ponderada
  • Projeção original: Mollweide (equal-area) → reprojeta para WGS84 na consulta
  • Profundidades compostas: Média ponderada pela espessura. Ex: 0–30cm = (0–5cm × 5/30) + (5–15cm × 10/30) + (15–30cm × 15/30)
4. Contexto Geológico — SGB/CPRM

A partir da versão 2.0, a API consulta automaticamente o Serviço Geológico do Brasil (SGB, antigo CPRM) para obter a formação geológica do ponto.

CampoDescrição

lithologyRocha dominante (arenito, basalto, argilito, etc.)
geological_ageEra geológica (Mesozoico, Cenozoico, etc.)
sourceIdentificação da fonte e escala
Por que isso importa

A formação geológica determina o mineral de argila dominante, que determina o comportamento mecânico:

MineralEstruturaComportamentoOnde ocorre
Caulinita (1:1)Lamelas rígidas, sem expansãoInerte. Argila não contribui para plasticidade, expansão ou retençãoGrupo Caiúa/Bauru (MS, PR, SP), Barreiras (NE), Latossolos do Cerrado
Montmorillonita (2:1)Lamelas expansivas, absorve águaAtiva. Causa expansão, alta plasticidade, recalques diferenciaisGuabirotuba (Curitiba), solos aluviais, vertissolos
MistaCombinaçãoParcialmente ativaSerra Geral (basalto), sedimentos marinhos
Sem essa correção, todas as PTFs erram sistematicamente. O SoilGrids reporta % de argila total sem distinguir mineralogia. 31% de caulinita se comporta como ~10% de argila ativa. 31% de montmorillonita se comporta como ~47%.
Fallback

Se o SGB estiver indisponível (timeout 5s ou erro), a API funciona normalmente sem correção — retorna dados brutos e derivações sem ajuste, com confidence_level: "low".

6. Derivações Geotécnicas (10 PTFs)

Todas as funções são pure Python (zero dependência de GEE) implementadas em geotech.py.

6.1 Condutividade Hidráulica (Ksat) Saxton & Rawls 2006
Input: areia (%), argila efetiva (%), matéria orgânica (%)Output: Ksat em mm/h + classe

Estima a condutividade hidráulica saturada via regressão multivariada calibrada em ~2.500 amostras (USDA). Calcula umidade na capacidade de campo, ponto de murcha, saturação, distribuição de poros e Ksat = 1930 × (θs − θ33)^(3 − λ).

6.3 Limites de Atterberg Regressão PTF
Input: argila efetiva (%), SOC (g/kg), CEC (cmol/kg)Output: LL, PL, PI estimados

Estima os limites de consistência via regressão linear:

LL = 0.57 × argila + 0.29 × CEC + 2.0 × OC% + 10
PL = 0.35 × argila + 0.12 × CEC + 1.5 × OC% + 5
PI = LL − PL  (mínimo 0)
Limitação: A regressão não modela o conceito de “não-plástico” (NP). Mesmo com argila efetiva baixa, CEC e SOC contribuem para um PI residual.

6.8 Avaliação de Fundação Regras compostas
Input: grupo USCS, shrink-swell, CBR, BDOutput: viabilidade, tipos recomendados, cautelas

Lógica baseada em regras:

  • Fundação rasa viável se: shrink-swell ≤ Moderate, CBR ≥ 3, USCS não-orgânico, BD ≥ 1.2
  • Tipos recomendados (rasa): Sapata corrida, Radier, Sapata isolada, Tubulão a céu aberto
  • Se não viável: Estaca cravada, Estaca hélice contínua, Tubulão, Radier estaqueado
6.9 Comparação de Camadas Topsoil vs Subsoil
Input: topsoil (0–30cm), subsoil (30–100cm)Output: salto de argila, salto de BD, impedimento, notas
  • Salto de argila > 15% → horizonte B textural (impedimento)
  • Salto de BD > 0.2 kg/dm³ → camada densa/compactada (impedimento)
  • Impedimentos → risco de lençol freático suspenso, drenagem deficiente

Checklist de Inspeção (exemplos)
  • Flag “permeabilidade” → Verificar projeto de drenagem — solo de baixa permeabilidade
  • Flag “erodibilidade” → Verificar medidas de controle de erosão (proteção de taludes, drenagem)
  • Flag “plasticidade” → Confirmar limites de Atterberg com ensaio de laboratório
  • Flag “expansão” → Monitorar movimentação sazonal do solo — risco de solo expansivo

10. Modelo de Complementaridade

A API não substitui o campo. A API direciona o campo.

Camada 1 — Triagem Remota (API)
Entrada: Coordenada do terreno
Saída: Perfil de solo + geologia + derivações + risco
Tempo: ~5 segundos
Custo: Zero
  • Go / no-go inicial (risk score alto → rejeitar ou investigar)
  • Precificação preliminar do prêmio (premium_factor)
  • Tipo de inspeção necessária (padrão vs reforçada vs completa)
  • Equipamento que o inspetor deve levar
  • O que o inspetor deve testar prioritariamente
Camada 2 — Inspeção Dirigida (Campo)
Entrada: Relatório da Camada 1 + terreno físico
Saída: Laudo geotécnico (SPT, lab, penetrômetro)
Tempo: 1–4 semanas
Custo: R$ 5.000–50.000
  • Inspetor chega preparado (sabe a formação, litologia, mineral)
  • Sabe o que validar (flags + checklist específico)
  • Sabe o que não testar (sem flag → simplifica)
  • Foco em parâmetros que a API não mede (SPT, NA, colapso)
Camada 3 — Laudo Final (Decisão)
Entrada: Camada 1 + Camada 2
Saída: Aprovação/rejeição do seguro + prêmio definitivo
Responsável: Engenheiro geotécnico (CREA)
  • Dados remotos (contexto, tendência, screening)
  • Dados de campo (SPT, lab, oedômetro)
  • Experiência do engenheiro
  • Conformidade regulatória (NBR 6122, NBR 8036)
Cenário A (sem API) vs Cenário B (com API)
Cenário A — SEM API
  1. Recebe endereço do terreno
  2. Envia equipe de campo sem informação prévia
  3. Inspetor faz investigação genérica (SPT padrão)
  4. Laboratório analisa tudo
  5. Engenheiro elabora laudo
  6. Underwriting analisa e precifica

Problemas: Inspeção genérica, sem priorização, custo uniforme, inspetor sem contexto.

Cenário B — COM API
  1. Recebe endereço do terreno
  2. API retorna em 5s: formação, mineral, risco, confiança
  3. Underwriting faz pré-aprovação condicional
  4. Envia inspetor com briefing dirigido
  5. Inspeção focada (mais rápida, mais barata)
  6. Laudo final ajusta prêmio se necessário

Ganhos: Inspetor preparado, escopo reduzido, custo menor, decisão mais rápida.

11. Limitações Conhecidas
Do SoilGrids (dados brutos)
  • Não diferencia mineralogia (corrigido parcialmente pela integração CPRM)
  • Resolução 250m — pode misturar solos de unidades diferentes num mesmo pixel
  • Profundidade máxima 200cm — insuficiente para perfil de fundação (SPT alcança 18m+)
  • Sem dados mecânicos (SPT, CPT, nível d'água, colapsividade, agressividade)
  • Acurácia variável — pior em tropicais intemperizados
Das PTFs (derivações)
  • Atterberg não modela NP — CEC e SOC inflam PI mesmo com argila efetiva baixa
  • Saxton-Rawls: otimizado para solos com finos; subestima Ksat em areias limpas
  • CBR regressão: subestima para areias com BD real elevado (aterro compactado)
  • USCS aproximado — sem ensaio de granulometria completa
Da integração CPRM
  • Serviço público instável — timeout possível (fallback sem correção)
  • Escala 1:1M — transições geológicas < 1km podem não ser representadas
  • Tabela com ~25 formações; não mapeadas usam ratio 1.0
Estruturais (nunca vai medir)
  • Resistência à penetração (SPT/CPT) — requer ensaio de campo
  • Nível d'água — requer sondagem
  • Colapsividade — requer oedômetro com inundação
  • Agressividade ao concreto — requer análise química
  • Recalques — requer modelagem com dados de campo
